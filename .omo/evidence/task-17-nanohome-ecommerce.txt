Implementation notes
- src/app/api/cron/amis-sync/route.ts exports POST(request: Request), reads Authorization, accepts only Bearer ${CRON_SECRET}, returns 401 on missing/wrong secret, and returns 200 for authenticated sync outcomes.
- src/lib/amis/client.ts uses a typed AMIS client boundary with Zod parsing. It fetches GET /api/v1/inventory/items with limit=200, optional updated_since watermark, X-API-Key, and optional X-Tenant.
- src/lib/amis/sync.ts creates a running amis_sync_log row, reads latest success/partial watermark, fetches one AMIS page, updates variants by SKU only, and finalizes the log as success/partial/failed with counts, error, and watermark.
- Variant writes are limited to planned fields: price, compare_at_price, discount_percent, in_stock, source_updated_at.
- Missing AMIS_API_BASE_URL or AMIS_API_KEY returns 200 failed and logs the failure where possible. AMIS_CLIENT_ID, AMIS_CLIENT_SECRET, and AMIS_TENANT are not enough without an auth/token/HMAC pattern.
- Auth pattern risk: implemented conservative static API key header X-API-Key. If AMIS expects OAuth/client-credentials, HMAC, bearer token exchange, or tenant-scoped endpoint shape, provide the exact AMIS API auth documentation and endpoint contract.
- vercel.json schedules /api/cron/amis-sync every 15 minutes.
